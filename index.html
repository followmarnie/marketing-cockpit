So funktioniert das Cockpit

Drei Module, eine Kette: Du skizzierst den Weg (Funnels) → jeder Schritt bekommt seinen Namen + UTM (UTM & Naming) → die Messpunkte stehen im Katalog (Conversions). Die Klicks fließen mit den UTMs automatisch in GA4; das Reporting zeigt Looker Studio.

Modul 1 · UTM & Naming — in 3 Schritten

  1. Plattform-Typ wählen — Werbe (Meta/Google/…), E-Mail (Active Campaign) oder Organisch. Die Felder passen sich an.
  2. Felder ausfüllen — nur SOP-erlaubte Werte (Dropdowns). Rechts entstehen live: Campaign-Name, AdSet/Automation, Identifier und alle utm_*-Parameter (bei Google-Search auch utm_term).
  3. „In Datenbank speichern" — die utm_id (z. B. tradeneon00050) vergibt die Datenbank automatisch und kollisionsfrei. Die volle URL landet in der Registry und in Deiner Zwischenablage.

Registry pflegen

  • ✎ Zeile inline bearbeiten (✓ speichert) · ⧉ URL kopieren · ✕ löschen (mit Rückfrage).
  • Filter Jahr/Kampagne/Source/Funnel · CSV ⬇/⬆ als Backup/Import · utm_id-Override nur für bewusste Ausnahmen.

Modul 2 · Funnels — den Weg skizzieren

  • Ein Funnel = eine Kette von Schritten (Anzeige → LP → Formular → E-Mail → Danke-Seite …).
  • Pro Schritt: Typ, Titel, URL, optional der Conversion-Messpunkt (verlinkt Modul 3) und eine Notiz.
  • Der UTM-Chip am Schritt zählt, wie viele Registry-Links auf die Schritt-URL zeigen — so siehst Du, was schon getaggt ist.
  • Schritte mit ↑↓ ordnen, ✕ entfernen, „+ Schritt" ergänzen — dann Speichern.

Modul 3 · Conversions — der Messpunkt-Katalog

  • Je Conversion: GA4-Event, Google-Ads-Label + Wert, Meta-Event — mit Status (✓ verdrahtet, offen).
  • Das ist die Abstimm-Fläche mit dem Tracking (Praba): Was messen wir wo, und ist es live?
  • Neue Conversion unten anlegen; Details per ✎ bearbeiten.

Wichtige Regeln

  • Interne Links nie taggen (Menü, Footer) — sonst überschreibt der interne Klick die echte Quelle in GA4.
  • Neue Source/Medium-Werte erst in SOP_Platform_Naming.md aufnehmen — das Tool warnt.
  • Regelquelle bleibt die SOP; das Tool bildet sie nur ab.

How the Cockpit works

Three modules, one chain: sketch the path (Funnels) → every step gets its name + UTM (UTM & Naming) → the measuring points live in the catalog (Conversions). Clicks flow into GA4 automatically via the UTMs; reporting happens in Looker Studio.

Module 1 · UTM & Naming — 3 steps

  1. Pick the platform type — Advertising, Email (Active Campaign) or Organic. Fields adapt.
  2. Fill the fields — SOP-allowed values only (dropdowns). Live output: campaign name, ad set/automation, identifier and all utm_* parameters (incl. utm_term for Google Search).
  3. “Save to database” — the utm_id (e.g. tradeneon00050) is assigned automatically and collision-free. The full URL lands in the registry and your clipboard.

Manage the registry

  • ✎ edit a row inline (✓ saves) · ⧉ copy URL · ✕ delete (with confirmation).
  • Filters year/campaign/source/funnel · CSV ⬇/⬆ backup/import · utm_id override for deliberate exceptions only.

Module 2 · Funnels — sketch the path

  • A funnel = a chain of steps (ad → LP → form → email → thank-you page …).
  • Per step: type, title, URL, optionally the conversion measuring point (links module 3) and a note.
  • The UTM chip on a step counts how many registry links point at that URL — so you see what's tagged.
  • Reorder with ↑↓, remove with ✕, add via “+ Step” — then Save.

Module 3 · Conversions — the measuring-point catalog

  • Per conversion: GA4 event, Google Ads label + value, Meta event — with status (✓ wired, open).
  • This is the alignment surface with tracking (Praba): what do we measure where, and is it live?
  • Add new conversions below; edit details via ✎.

Key rules

  • Never tag internal links (menu, footer) — the internal click would overwrite the real source in GA4.
  • Add new source/medium values to SOP_Platform_Naming.md first — the tool warns you.
  • The SOP stays the rule source; the tool only reflects it.
