So funktioniert das Cockpit

Drei Tabs, eine Kette — genau wie die SOP es beschreibt: Du benennst die Kampagne (Naming) → Du baust die Tracking-URL dazu (UTM-Builder) → Du skizzierst den Weg und stimmst die Messpunkte ab (Funnels & Conversions).

1 · Naming — der Kampagnen-Namen-Generator

  • Felder ausfüllen (nur SOP-erlaubte Werte). Rechts entstehen live: Campaign Name, AdSet/Automation, Ad-/E-Mail-Identifier und die abgeleiteten utm_campaign + utm_content.
  • „Namen speichern" legt eine Zeile in der Namensliste an — eine pro Ad/E-Mail. Diese Namen benutzt Du im Ads Manager / in Active Campaign.
  • In die spätere URL fließt nur der Campaign Name (als utm_campaign). Der AdSet-Name hat bewusst kein UTM-Pendant.

2 · UTM-Builder — die Tracking-URL

  • Oben einen Naming-Eintrag wählen → utm_campaign und utm_content werden übernommen. Oder manuell tippen.
  • Source und Medium als Dropdown wählen (Medium füllt sich automatisch passend zur Source, bleibt aber änderbar), Funnel-Referenz zuordnen, Ziel-URL eintragen.
  • „In Registry speichern" → die utm_id (z. B. tradeneon00050) vergibt die Datenbank automatisch, die volle URL landet in Deiner Zwischenablage.
  • Registry: ✎ inline bearbeiten · ⧉ URL kopieren · ✕ löschen · Filter & CSV.

3 · Funnels & Conversions — der Weg + die Messpunkte

  • Funnels als Liste: Klick auf den Namen öffnet den Schritt-Editor (Anzeige → LP → Formular → … → Danke). Funnel archivieren statt löschen, wenn er nur ruht.
  • Ein Schritt kann auf einen Conversion-Messpunkt zeigen; darunter fächert auf, was Praba je Kanal (GA4/Ads/Meta) verdrahten muss.
  • Conversions darunter als Liste: Praba pflegt hier den Status (✓ verdrahtet, offen).

Wichtige Regeln

  • Interne Links nie taggen (Menü, Footer) — sonst überschreibt der interne Klick die echte Quelle in GA4.
  • Neue Source/Medium-Werte erst in SOP_Platform_Naming.md aufnehmen.
  • Regelquelle bleibt die SOP; das Tool bildet sie nur ab.

How the Cockpit works

Three tabs, one chain — exactly as the SOP describes: you name the campaign (Naming) → you build the tracking URL (UTM builder) → you sketch the path and align the measuring points (Funnels & Conversions).

1 · Naming — the campaign name generator

  • Fill the fields (SOP-allowed values only). Live output: campaign name, ad set/automation, ad/email identifier and the derived utm_campaign + utm_content.
  • “Save name” adds a row to the name list — one per ad/email. You use these names in the ads manager / Active Campaign.
  • Only the campaign name flows into the later URL (as utm_campaign). The ad-set name deliberately has no UTM counterpart.

2 · UTM builder — the tracking URL

  • Pick a naming entry at the top → utm_campaign and utm_content are carried over. Or type manually.
  • Choose source and medium from dropdowns (medium auto-fills to match the source, stays editable), assign a funnel reference, enter the target URL.
  • “Save to registry” → the utm_id (e.g. tradeneon00050) is assigned automatically, the full URL lands in your clipboard.
  • Registry: ✎ edit inline · ⧉ copy URL · ✕ delete · filters & CSV.

3 · Funnels & Conversions — the path + the measuring points

  • Funnels as a list: click the name to open the step editor. Archive a funnel instead of deleting when it only rests.
  • A step can point at a conversion measuring point; below it fans out what Praba has to wire per channel (GA4/Ads/Meta).
  • Conversions below as a list: Praba maintains the status here (✓ wired, open).

Key rules

  • Never tag internal links (menu, footer) — the internal click would overwrite the real source in GA4.
  • Add new source/medium values to SOP_Platform_Naming.md first.
  • The SOP stays the rule source; the tool only reflects it.
